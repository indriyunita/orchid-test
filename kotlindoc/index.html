How to Commit

  • Make sure your commit message is easy to understand
  • Commit message format [YOUR_NAME] your detail commit message
  • If possible, use only 1 push, but it's ok with multiple commits
  • Always sync with the branch you want to merge to, for example in your branch feature/feature-name : git pull --rebase origin master, will sync your branch with master and make sure to run this after commit & BEFORE you push code

Gradle setup

  • dependencies.gradle store a list of all used dependencies, if you want to add more dependencies, make sure to check the source whether your lib is still well maintained or not
  • dokka-setup.gradle used for generate javadocs from our core (it's not magical, we still need to add comment info to be generated as javadocs)

Our friendly spell

  • ./gradlew dokka : to generate javadocs, the html output file will reside in : core/build/javadoc/core/index.html
  • ./gradlew core:assembleRelease --parallel --stacktrace --no-daemon -x lint : to assemble our core
  • ./gradlew core:artifactoryPublish --stacktrace --no-daemon -x lint : to manually deploy to our Artifactory website

Version?

  • Version number manually set in gradle.properties
